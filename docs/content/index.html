Pushup web framework

A proudly server-side, page-oriented web framework for the Go programming language.

Pushup’s goal is to make it faster to develop and easier to maintain server-side web applications using Go.

^import "time"

^{
   title := "Hello, from Pushup!"
}

<h1>^title</h1>

<p>The time is now ^time.Now().String().</p>

^if time.Now().Weekday() == time.Friday {
    <p>It's Friday! Enjoy the start to your weekend.</p>
} ^else {
    <p>Have a great day, we're glad you're here.</p>
}

Features

  • Pages: .up files

    Flexible units of web app development, combine HTML, Go, and Pushup markup

  • File-based routing

    Filesystem paths of Pushup pages map to URL paths

  • World’s first ‟htmx-first” framework

    Enhanced hypertext support via inline partials for better client-side interactivity with fewer JavaScript sit-ups

  • Compiled. Just Go.

    Pushup apps compile down to pure Go. They build on the standard net/http package. Fast static binary executables for easy deployment. Also easy to integrate into larger Go apps.

What is Pushup?

Pushup is actually three things in one:

  1. A lightweight markup syntax

    Pushup markup glues HTML and Go together, using the ^ (caret, ASCII 0x5E) character as a transition between the two.

  2. An opionated app project layout

    Pushup apps have a simple directory structure on disk for easy project file management.

  3. A compiler

    The pushup program compiles Pushup files (.up extension) to Go.
